Projet Veliko - DevOps

Projet Veliko - Gestion des Vélos Urbains

Veliko est une application permettant aux utilisateurs de suivre la disponibilité des vélos en libre-service dans le cadre du système Vélib' Métropole à Paris. Avec un réseau de 1 464 stations réparties à travers Paris et l'Île-de-France, l'application fournit des informations en temps réel sur la disponibilité des vélos, y compris la distinction entre vélos classiques et vélos électriques.

Mon Rôle - DevOps

En tant que responsable DevOps, mon rôle a été essentiel dans la mise en place de l'infrastructure, la gestion de l'automatisation des déploiements, et la sécurisation des serveurs pour garantir que l'application fonctionne de manière optimale et sécurisée. J'ai contribué à la mise en place des bonnes pratiques DevOps pour faciliter les mises à jour et garantir la scalabilité et la sécurité de l'application.

Phases du Projet

Release 1 : Préparation des Environnements de Développement

La première phase a consisté à préparer les environnements de développement pour les équipes. Il s'agissait de configurer les outils nécessaires pour la gestion de code source, l'intégration continue, et la validation des premiers développements. Cette étape a assuré une collaboration fluide entre les équipes et un environnement stable pour tester les premiers composants de l'application.

Release 2 & 3 : Mise en place de l'Infrastructure Serveur

Les releases 2 et 3 ont été centrées sur le déploiement de l'infrastructure nécessaire à l'application web. Voici les étapes clés de ces releases :

  • Machine Virtuelle : Mise en place d'une machine virtuelle (ESX ou similaire) pour héberger l'infrastructure, permettant une gestion flexible des ressources.
  • Serveur HTTP : Installation d'un serveur HTTP (Apache ou Nginx) avec PHP 8.2 et les modules nécessaires pour garantir le bon fonctionnement de l'application web.
  • Serveur de Base de Données : Mise en place d'un serveur de base de données (par exemple MySQL ou PostgreSQL) pour stocker les informations sur les stations, les vélos et les utilisateurs. Ce serveur assure la persistance des données essentielles pour le bon fonctionnement de l'application.
  • Serveur SFTP : Mise en place d'un serveur SFTP pour la livraison sécurisée de l'application et des mises à jour régulières. Cela permet un transfert de fichiers sécurisé entre les développeurs et le serveur de production.
  • Domaine Interne : Configuration d’un domaine interne sous la forme gr.veliko.lan pour faciliter l’accès sécurisé et l'administration des ressources sur le réseau local.

Release 4 : Sécurisation de l'Architecture et Automatisation

La phase finale a porté sur la sécurisation de l'infrastructure et l'automatisation des processus afin de garantir la stabilité à long terme. Voici les actions clés :

  • Gestion des Backups : Mise en place d'une stratégie de sauvegarde de la base de données incluant des sauvegardes complètes et incrémentielles. Des tests réguliers de restauration ont été effectués pour assurer la fiabilité des backups.
  • Sécurisation de l'Architecture : Application des recommandations de l'ANSSI pour sécuriser les serveurs et les communications. Cela inclut la configuration de pare-feu, l’utilisation de VPN pour sécuriser les échanges internes, et l’audit des systèmes pour garantir qu’ils sont protégés contre les attaques externes.
  • Sécurisation du Code : Mise en place de processus rigoureux pour la revue de code et l’audit de sécurité. Cela a permis de détecter et corriger les vulnérabilités, comme les injections SQL et les failles de sécurité potentielles dans le code.
  • Automatisation du Déploiement : Mise en place d’un pipeline CI/CD (Continuous Integration / Continuous Deployment) avec des outils comme Jenkins ou GitLab CI. Cela a permis d’automatiser les tests, la validation et le déploiement de nouvelles versions de l’application sur les environnements de staging et de production. Cette automatisation a assuré des mises à jour rapides et sécurisées de l’application.

Conclusion

Le projet Veliko a permis de mettre en place une infrastructure robuste, sécurisée et scalable pour supporter une application web complexe dédiée à la gestion des vélos en libre-service. Grâce à une approche DevOps, nous avons optimisé le processus de développement, automatisé les déploiements, et assuré la sécurité des données et des systèmes. Ce projet est un excellent exemple de l'importance de l'automatisation et de la sécurisation dans les environnements de production modernes.
